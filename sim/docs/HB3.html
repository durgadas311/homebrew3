Running

The typical invokation is: 

java -jar VirtualHomeBrew3.jar <config-file>

In addition, the keyword "gui" can be added to enable the GUI, or that may be put as a property in the config-file.

Properties

log = <file> [a]
Send stderr of simulator to file. 'a' after filename causes append instead of overwrite.
debugger
Enables debugger input on stdin. Output is to stderr.
disas = <mnemonics>
Select disassembly mnemonics. 'zilog' or 'MAC80' (Intel with Z80.LIB).
con_att = <attach-spec>
Specify connection to SIO-A.
sys_att = <attach-spec>
Specify connection to SIO-B.
trace = <trace-command>
Setup trace command before starting CPU.
monitor_rom = <file>
Specify file for monitor ROM image. Default is "homebrew3.rom".
gui
Enables GUI.
display_test = <string>
Specifies initial contents of display (if GUI enabled).
switches = <binary>
Selects initial positions of toggle switches. LSB in left-most switch (4800/9600).

Attach Specifications

> <file> [a] [+]
Direct output to file (no input). 'a' appends if file exists. '+' selects exclusive (no other device attachments get output). Special name 'syslog' directs output to system log (stderr).
TelnetSerial <host> <port> [modem] [nodtr]
Make a telnet-like connection to host at port. 'modem' enables modem controls following connection status. 'nodtr' disables hang-up on loss of DTR.
ProgramSerial <command-line>
Connect serial port to stdio of command-line.

Debugger Commands

Debugger commands are entered on stdin to the JAVA process. Output is sent to stderr, which may be redirected to a file using the "log" property.
quit
Exit the simulator. This is the same as the 'quit' option on the GUI menu or clicking the close button on the GUI window.
reset
Perform a RESET. This is the same as the 'reset' option on the GUI menu, or clicking the RESET button on the GUI.
nmi
Perform an NMI. This is the same as the 'nmi' option on the GUI menu, or clicking the NMI button on the GUI.
trace <trace-command>
Trace commands:
on
Turn on tracing of all CPU instructions executed.
off
Turn off whatever type of tracing is enabled.
cycles <number>
Trace the number of CPU cycles.
pc <low-pc> [<hi-pc>]
Trace all instructions between low-pc and hi-pc (exclusive). The default hi-pc is 0x10000.
dump <dump-command>
Dump commands:
page <hex-num>
Dump the 256-byte memory page.
cpu
Dump information about the CPU.
mach
Dump information about the basic machine.
dev <dev-name>
Dump debug information about the device. Names are case-sentitive.
getdevs
Lists the peripheral device names that may be dumped with the "dump dev..." command.

In general, the "Debug" menu provides access to the same debugger commands. By default, commands from this memu show results in a pop-up window. An option exists to redirect to the "log" (stderr).

The "System" menu provides Reset, NMI, and Quit commands.
